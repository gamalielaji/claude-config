# Revenue Protocol Reference

## The Revenue Gate System

Every project must pass ALL 5 gates before getting Claude time:

### Gate 1: Revenue Connection
- Question: "How does this generate or protect revenue?"
- PASS: Clear thesis in 1 sentence
- FAIL: "I just think it's interesting" / "it might be useful someday" → Park or kill

### Gate 2: Revenue Timeline
- Question: "When does this produce money?"
- PASS: This week / this month / this quarter / this year (specific)
- FAIL: "Eventually" / "when it's ready" / no timeline → Not a project yet

### Gate 3: Slot Available
- Question: "Is there room?" (Max 3 active projects)
- PASS: <3 active → Proceed
- FAIL: 3 active → Ship or kill one first. No exceptions.

### Gate 4: Session Budget
- Question: "How long does this take?"
- Small: 1 session (email, deck, one-pager, brief)
- Medium: 2-3 sessions (strategy doc, skill, dashboard, proposal)
- Large: 5 sessions max (full system, multi-doc, complex build)
- FAIL: "I don't know" → Scope is unclear. Define deliverable first.

### Gate 5: Owner Assigned
- Question: "Who builds, reviews, ships?"
- PASS: Clear ownership chain
- FAIL: "I'll figure it out" → Assign now or don't start

## Revenue Classification Deep Dive

### R1: Direct Revenue
**Definition:** Work that directly results in a signed deal, invoice, or cash receipt.
**Examples at INFIA:**
- Client pitch deck customized for a specific brand
- Proposal with pricing for a campaign
- Contract negotiation support
- Rate card presentation
- Follow-up on warm lead

**Behavioral rule:** If you don't have an active R1 project, STOP EVERYTHING ELSE until you do. R1 is oxygen.

### R2: Pipeline Building
**Definition:** Work that creates the conditions for future R1 conversions.
**Examples at INFIA:**
- Credentials deck refresh (DAGELAN MEMBUAT LAYAK VIRAL)
- CRM pipeline activation and outreach
- FMCG contact database building
- Case study production
- Networking/relationship prep
- Content that demonstrates capability to potential clients

**Behavioral rule:** R2 is the investment that pays off in R1. Always have at least 1 R2 active.

### R3: IP / Asset Building
**Definition:** Creating durable assets that generate recurring or compounding revenue.
**Examples at INFIA:**
- IP licensing frameworks
- Digital product development
- Content IP creation (Tahilalats global expansion, new IP)
- Brand equity building (Dagelan lore, cultural positioning)
- Platform features that create lock-in

**Behavioral rule:** R3 is the $1B play. But it doesn't pay rent. Max 1 of 3 slots. Never at the expense of R1.

### R4: Infrastructure
**Definition:** Systems that reduce cost, increase speed, or multiply output of R1-R3.
**Examples at INFIA:**
- Data ops standardization (templates, validators)
- Automation workflows
- Claude skills and knowledge systems
- Team process documentation
- Performance reporting automation

**Behavioral rule:** R4 is seductive for ADHD brains because building systems is stimulating. But infrastructure without revenue to flow through it is a hobby. Only invest in R4 when R1/R2 pipeline is healthy.

## The Revenue Scoreboard

Claude tracks across conversations:
- Projects shipped this month: [N]
- Revenue generated/attributed: [Rp X]
- Projects killed: [N]
- Active projects: [N/3]
- Revenue mix: R1 [%] / R2 [%] / R3 [%] / R4 [%]

The real performance metric: **Projects shipped × revenue impact.** Not hours worked. Not projects started.
