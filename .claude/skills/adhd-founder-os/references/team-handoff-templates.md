# Team Handoff Brief Templates

## Universal Template

Every handoff Claude generates includes these 7 sections:

```
1. REVENUE THESIS: [Why this matters commercially — 1 sentence]
2. DELIVERABLE: [Exactly what they produce — specific, measurable]
3. CONSTRAINTS: [Budget, deadline, format, brand guidelines]
4. DECISIONS MADE: [What's locked — do NOT re-open these]
5. REFERENCE: [Examples of "good" — links, screenshots, past work]
6. REVIEW CHECKPOINT: [When you'll review + what you'll check for]
7. SHIP DATE: [Non-negotiable. Build backward from this.]
```

The brief IS the management. ADHD founders can't micromanage — the document does it.

## Team-Specific Templates

### BD / Sales (Konner Advisory)

**Delegate:** Meeting execution, client relationship management, proposal delivery, follow-up cadence
**Keep:** Deal structure, pricing strategy, partnership terms, strategic positioning

Prompt: "Build BD package for [brand]:
1. Custom proposal with pricing tiers from INFIA Solutions rate card
2. Credentials deck customized to their vertical
3. Meeting talking points (5 max)
4. Objection handlers for top 3 likely pushbacks
Revenue thesis: [X]. Meeting date: [Y]. Ship brief by: [Z]."

### Content / Social Team

**Delegate:** Content production, scheduling, platform optimization, community response
**Keep:** Editorial direction, what's "layak viral", cultural judgment, trend selection

Prompt: "Build content execution package for [period]:
1. Editorial calendar with post dates
2. Per-post briefs: hook + angle + CTA + visual direction
3. Platform-specific specs (IG feed vs story vs reels vs TikTok)
4. Performance benchmarks to hit
Revenue thesis: this pipeline feeds [X] brand conversations / [Y] R2 leads."

### Creative / Design Team

**Delegate:** Visual execution, motion graphics, UI polish, brand consistency, production files
**Keep:** Creative direction, narrative angle, what matches brand truth

Prompt: "Creative brief for [project]:
1. Objective (what this asset achieves)
2. Audience (who sees it, in what context)
3. Tone + visual mood (with 3 reference examples)
4. Deliverable specs (dimensions, format, file types)
5. Brand constraints (what NOT to do)
Revenue thesis: this asset supports [X deal / product / campaign]."

### Dev / Tech Team

**Delegate:** Implementation, deployment, testing, infrastructure setup
**Keep:** Product vision, feature priority, UX judgment, architecture decisions

Prompt: "Technical spec for [feature]:
1. User stories (as a [user], I want [X] so that [Y])
2. Acceptance criteria (specific, testable)
3. Data model / API endpoints if applicable
4. Edge cases and error handling
5. Dependencies and blockers
Revenue thesis: this feature enables [X revenue stream]. Ship date: [Y]."

### Data / Analytics Team

**Delegate:** Dashboard building, report automation, data cleaning, metric tracking
**Keep:** What to measure, what the data means, strategic implications

Prompt: "Data requirements doc:
1. Metrics needed (with exact definitions)
2. Data sources + access requirements
3. Update frequency (real-time / daily / weekly)
4. Report format + who receives
5. Alert thresholds (when to flag)
Revenue thesis: this data reduces decision latency for [X revenue decisions]."

## The Handoff Flow

```
YOU (decide what + why) → CLAUDE (builds the brief) → TEAM (executes) → YOU (reviews against brief) → SHIP
```

Rules:
- The brief is the contract. If it's not in the brief, it wasn't requested.
- Review against the brief, not against your evolving ideas. Scope creep in review is as deadly as scope creep in build.
- If the deliverable matches the brief, it ships. Don't move the goalposts.
- If you realize the brief was wrong, that's a new brief (new session, new scope), not a revision of the current work.
